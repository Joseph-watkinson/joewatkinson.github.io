TLS 1.2 RSA Key Exchange Decryption

Purpose

This project tested the vulnerability of TLS 1.2 sessions using RSA key exchange. The goal was to show how a compromised private key allows attackers to decrypt captured traffic, even after the session has ended due to the lack of forward secrecy in RSA ciphers.

Methodology

Three VMs were configured: a client, an attacker, and an nginx webserver with a self-signed certificate. The server was restricted to RSA cipher suites under TLS 1.2. The client logged into a simple HTTPS form hosted by the webserver, while the attacker monitored traffic with tcpdump.

Process

Traffic was captured with tcpdump and saved as a .pcap. In Wireshark, HTTP filters confirmed TLS 1.2 with RSA.

Result

Importing the server’s private key into wireshark enabled decryption of the ClientHello and subsequent packets.Subsiquently, the HTTP POST request, including plaintext username and password, was successfully recovered.

Value

Demonstrates setup of a secure lab, traffic capture and decryption, and highlights the weakness of TLS 1.2 RSA compared to forward-secret key exchange.
